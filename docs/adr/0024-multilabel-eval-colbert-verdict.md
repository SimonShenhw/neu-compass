# ADR-0024: 多标签评测 (test_set v0.4) + ColBERT 迁移否决

## 状态

Accepted - 2026-06-12 (v0.4 为 canonical 评测集;ColBERT 迁移否决,
重启条件见后果节)

## 背景

v0.2-v0.3.1 的 ground truth 是单标签 weak label(生成查询只认源课程)。
q108 暴露了后果:系统返回"另一门同样正确的课"被记 0 分 — 评测假阴性
既低估质量,又可能否决好改动。medium 类 0.644 的"天花板"因此存疑,
而它是 ColBERT 架构迁移论证的核心依据 — **必须先修尺子再动系统**。

## 方法 (scripts/build_test_set_v04.py, UMBRELA 式池化标注)

1. **池化**:每条可答查询取 union(原 expected, 生产 live top-10) —
   对部署系统池化是 TREC 惯例(只标注检索实际能浮出的候选)
2. **标注**:每查询一次 Gemini listwise 调用,对池内全部候选打
   UMBRELA 0-3 分(按课程内容 vs 查询意图,明确禁止字面重叠打分)
3. **组装**:expected = {grade ≥ 2} ∪ {源课程 if grade ≥ 1};
   源课程被判 0 → 整条查询作废(生成质量问题,结果 0 条作废);
   adversarial 的 expected=[] 语义不动。judge_grades 全量存档在
   测试集文件里供审计
4. **度量配套**:recall@k 分母封顶 min(|expected|, k)(R-precision
   惯例)— 否则 12 标签查询的 top-5 上限只有 0.42。对旧单标签集是
   无操作,历史数字不受影响

结果:116 条,平均 3.92 标签/查询(max 12),8 条源课程判 1 分
(保留并标记)。

## 实测 (同一部署系统,仅换评分口径)

| 类别 | v0.3.1 单标签 | v0.4 多标签 R@5 / MRR |
|---|---:|---:|
| simple | 0.908 | 0.987 / 1.000 |
| medium | 0.644 | **0.762 / 0.886** |
| complex | 0.675 | 0.711 / 0.849 |
| boundary | 1.000 | 0.956* / 1.000 |
| code_switched | 0.917 | 0.967 / 0.938 |
| **整体** | 0.799 / 0.743 | **0.863 / 0.931** |

*boundary 微降非回归:多标签把 expected 扩大,分母变了。

medium 的修正幅度 (+0.118) 证实约一半"短板"是评测假阴性;且 v0.4 下
**没有任何 medium 查询全错**(R@5=0 集合为空),MRR 0.886 = 用户第一眼
基本就能看到相关课。

## 决定:ColBERT late-interaction 迁移 — 否决(当前证据下)

迁移论证建立在"medium 0.644 是检索架构瓶颈"上。修正后:

- 整体 MRR 0.931、medium MRR 0.886、零全错查询 — 痛点不存在
- 迁移成本未变:新索引管线、文档侧离线计算、英文模型(answerai-colbert-
  small)对中/混合查询还要另配多语版本、NAS 内存压力重估
- 收益上限被压缩:R@5 0.86 → 理论 1.0 的残余空间里,相当部分是
  "相关课多于 5 个名额"的展示问题,不是召回问题

**重启条件**:真实查询日志(query_log)显示用户对 NL 改写类查询的
点击位置系统性靠后,或 v0.5(真实分布)上 medium MRR 跌破 ~0.75。

## 后果

- v0.4 起为 canonical;历史 label 的数字对比要注明口径
  (单标签 vs 多标签 + 分母封顶)
- "先修尺子再动系统"二次验证:一天内第二次差点为测量伪影发起大改
  (第一次是 gate v3 的向量塌缩,这次是 ColBERT)
- 8 条 grade-1 源课程与全量 judge_grades 留档,真实点击数据进来后
  与 LLM 标注对账,校准 judge prompt
